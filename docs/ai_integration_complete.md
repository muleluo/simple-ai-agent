## 📋 集成内容

### 1. AI 客户端类 (`AIClient`)

从 `main.py` 迁移到 `main_qml.py`，包含：

- OpenAI 兼容的 API 调用
- 三个工具定义：
  - `set_reminder` - 设置定时提醒
  - `web_search` - 网络搜索
  - `search_knowledge_base` - 知识库搜索

### 2. 工作线程 (`ChatWorkerWithTools`)

实现了完整的工具调用流程：

1. 第一次调用 AI API（带工具列表）
2. 检查 AI 是否调用工具
3. 执行工具并获取结果
4. 将工具结果返回给 AI
5. AI 整理结果后返回给用户

### 3. 工具函数

- ✅ `search_web()` - DuckDuckGo 网络搜索（带重试机制）
- ✅ `knowledge_base.search()` - 知识库语义搜索
- ⏳ `set_reminder()` - 提醒功能（后端支持，待 UI 集成 QTimer）

### 4. PyBridge 更新

#### 新增属性
- `ai_client` - AI 客户端实例
- `system_prompt` - 系统提示词
- `worker` - 工作线程引用

#### 更新方法
- `sendMessage()` - 真实 AI 调用
  - 准备消息历史（最近 20 条）
  - 创建工作线程
  - 显示"正在思考"提示
  
- `on_ai_reply()` - 处理 AI 回复
  - 移除"正在思考"提示
  - 显示搜索提示（如果有）
  - 显示 AI 回复
  - 处理提醒（如果有）
  - 显示截断警告（如果有）

## 🚀 使用方法

### 1. 配置 API

编辑 `config.yaml`：

```yaml
api:
  api_key: "your-api-key-here"
  base_url: "https://api.openai.com/v1"
  model: "gpt-3.5-turbo"
  temperature: 0.7
  max_tokens: 1000
```

### 2. 启动应用

```bash
python3 main_qml.py
```

### 3. 测试功能

- **基础对话**：直接发送消息
- **联网搜索**：点击"···" → "联网搜索"，然后发送需要搜索的问题
- **知识库**：先上传文档到知识库，然后提问相关内容
- **提醒**：告诉 AI "5分钟后提醒我..."（UI 提示功能待完善）

## 🎨 用户体验

### 消息流程

```
用户输入消息
  ↓
显示用户消息气泡
  ↓
显示"AI 正在思考..."（系统消息）
  ↓
[后台] 调用 AI API
  ↓
[如果需要] 显示"🔍 正在搜索：关键词"
  ↓
移除"正在思考"提示
  ↓
显示 AI 回复气泡
  ↓
[如果有] 显示"⏰ 已设置提醒"
```

### 错误处理

- API 调用失败 → 显示错误消息
- 网络搜索失败 → 显示友好的错误提示
- 知识库为空 → 提示"未找到相关内容"

## 📊 技术细节

### 异步架构

```
主线程（QML UI）
    ↓
PyBridge.sendMessage()
    ↓
创建 ChatWorkerWithTools
    ↓
工作线程（后台）
    ├─ 调用 OpenAI API
    ├─ 执行工具（搜索/知识库）
    └─ 再次调用 API 整理结果
    ↓
发出 finished 信号
    ↓
PyBridge.on_ai_reply()
    ↓
更新 QML UI
```

### 消息上下文

- 系统提示词（始终包含）
- 最近 20 条历史消息
- 用户当前输入

### 工具调用

AI 可以主动调用工具：

```python
# AI 决定需要搜索
AI: 调用 web_search("最新新闻")
  ↓
# 执行搜索
结果: "1. 标题...\n2. 标题..."
  ↓
# AI 整理结果
AI: "根据搜索结果，最新新闻是..."
```
